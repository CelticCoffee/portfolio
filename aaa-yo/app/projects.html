Jot an intuitive scheduling app for small business

Login

We used Passport for user authentication, giving users the option of creating an account locally or signing in with Facebook.

Much of the design structure is derived from Bootstrap.

Dashboard

For the calendar, various options such as the Google Calendar API and the jQuery Clear Calendar were considered but none offered the look and lined up with the goals that we had for our project. At the forefront of the process was the user story of an individual with minimal computer skills. We wanted the app to be as easy to use as Facebook. Our calendar at its earliest stages was based off of Andrew Burgess’s tutorial on building a calendar widget however we used pure javascript and backbone in place of a lot of html hard coding.

Database and Framework

For the database, our app uses Orchestrate. Using Backbone for our JavaScript framework, we created collections and models for kids, bookings and calendar days. I especially enjoyed interacting with the javascript Date Object and as this was my first experience with Backbone, I learned a great deal along the way.

User Experience

We wanted as few windows to click through as possible so we opted to use modals for much of the booking processes in the app. A user can make, modify and delete bookings which are instantly reflected both on the visual calendar and in the list of bookings. We faced some inital challenges bringing the functionality we wanted together with our desired look but in the end, were able to get everything working smoothly. As we continue to improve and expand the app, we hope to put more of our code into handlebars templates. With just 3.5 weeks to build the app from start to deployment, we had to constantly balance taking time to learn all of the technologies out there with actually producing results.

Jot an intuitive scheduling app for small business

Jot is a full stack Javascript application. I contributed to every aspect of the stack. The current build is designed for a private school offering before and after school childcare. The app will be expanded to offer booking services to diverse small businesses.
Many business scheduling apps exist but most are geared for large corporations. Small businesses, particularly sole proprietors have neither the budget nor the time to implement these robust systems. Jot is different because it does one thing and it does it well: offers an intuitive scheduling interface to allow clients and small businesses to easily make and keep track of appointments.

Cathedral School

The ongoing reboot of Cathedral School’s website is a study in how to present heavy content in a way that is both elegant and easy to navigate. Busy parents visit the site to quickly gather important information but the site’s other purpose is to function as a primary marketing tool for the school. In addition to the technologies used on the programming side, I have had many meetings with faculty and parents to guide the direction of the site. The ability to listen and communicate with the client is as much of the process as the actual coding.
The site is nearing completion and will debut in early 2016.

Login

Passport is used for authentication, giving users the option of creating an account locally or signing in with Facebook. Password encription is achieved with Bcrypt.

Much of the design structure is derived from Bootstrap.

Dashboard

For the calendar, various options such as the Google Calendar API and the jQuery Clear Calendar were considered but with responsive design and a beautiful user experience at the forefront of our priority list, we opted to build the calendar in Javascript. The calendar at its earliest stages was based off of Andrew Burgess’s tutorial on building a calendar widget however we used vanilla javascript and backbone in place of a lot of html hard coding and ended up adapting the style to meet our needs. I especially enjoyed using the javascript Date Object.

Backbone Library

Using Backbone to bring organization to the project, we created collections, models and views. The data is stored on the backend in Orchestrate and we use Express for our server. Learning to write routes was a highlight of this experience.

User Experience

We wanted as few windows to click through as possible so we opted to use modals for much of the booking processes in the app. A user can make, modify and delete bookings which are instantly reflected both on the visual calendar and in the list of bookings.

Up to Date Information

Part of being a great developer and designer is knowing when to create from scratch and when to enlist third party tools. For the Cathedral home page, I embedded a google calendar Facebook feed to allow staff to easily keep the site up to date without needing to touch the code. Their current system involves administration modifying the actual website every time they want to highlight any current event. This saves time and money without taking any draw on the user experience.

Organize Content

The greatest challenge with this website is the vast amount of information that the administration wants to convey. I've organized this information into logical groupings with elegant design to help users quickly navigate to the information they need wihthout having to search through multiple windows and pages. Some of the structure I’ve achieved using javascript and jQuery to replace information once a menu item is clicked. Buttons, links and modals are also employed to achieve the organizational goals.

Feature Imagery

One of Cathedral School's selling points is its warm and welcoming atmosphere. I've placed responsive images throughout the site that tell that story without words. In the case of the school program page, I feature the images to give the user an overall impression that there is a lot of exciting activity going on at the school. jQuery activated buttons then expand out to offer the user detailed descriptions of the program offerings.

Mobile Friendly

The entire site is optomized for mobile phone use. This is so key when the organization’s primary target audience is likely going to be accessing the site more often from mobile than from a desktop. Bootstrap of course proved highly useful to this end but it also requires a careful consideration of the order of information since busy parents may not take the time to even scroll down a page. The information needs to be easy to find.
